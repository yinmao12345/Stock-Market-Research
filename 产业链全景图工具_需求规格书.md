# 产业链全景图工具 — 需求规格说明书（交付 Codex 实现）

> 用途：个人 A 股投研工具。以"细分产品产业链"为核心，逐级构建、检阅、拼接，形成可随时查阅、易分享、可迭代的全景拓扑图，辅助投资决策。

---

## 0. 一句话定位

一个**纯静态网页**（HTML + CSS + 原生 JS，无后端），数据存于 **结构化 JSON**，以**左右横向树状图**呈现产业链拓扑，节点带关键指标缩略、缺口状态用颜色编码、点击弹出侧边详情面板。可托管于 GitHub Pages，免费、易分享、离线可看。

## 2. 核心数据模型（JSON Schema）

这是整个工具的地基，请 Codex 严格按此实现。每个"产业环节"是一个节点对象，节点可嵌套子节点（上游），形成树。

```jsonc
{
  "id": "ccl",                          // 唯一标识，英文小写
  "name": "覆铜板 (CCL)",                // 显示名称
  "layer_updated": "2026-06-20",        // 该环节最后更新日期（环节层面统一记一个）

  // —— 节点上直接显示的关键指标缩略 ——
  "metrics": {
    "market_size": {                    // (1) 市场规模预测
      "value": 850,
      "unit": "亿元",
      "year": 2025,
      "note": "全球电子级CCL市场"
    },
    "cagr": {                           // (2) 未来3-5年复合增长率
      "value": 8.5,
      "unit": "%",
      "period": "2025-2030"
    },
    "value_share": {                    // (3) 价值量/成本占比（仅一级，相对其下游）
      "value": 30,
      "unit": "%",
      "relative_to": "PCB",             // 占谁的比，明确锚点
      "note": "CCL约占PCB成本30%"
    }
  },

  // —— 竞争格局：行业集中度 + 龙头市占率 + 梯队描述 ——
  "competition": {
    "concentration": {                  // 行业集中度，如 CR5 / CR10
      "label": "CR5",
      "value": 60,
      "unit": "%",
      "note": "统计口径、年份与来源"
    },
    "leader_share": {                   // 龙头市占率
      "company": "龙头公司名",
      "value": 25,
      "unit": "%",
      "note": "统计口径、年份与来源"
    },
    "tiers": "第一梯队A/B，第二梯队C/D，长尾厂商若干"
  },

  // —— (4) 缺口状态：底层指标 + 规则自动推导颜色 ——
  "gap": {
    "status": null,                     // 由规则自动推导，可留空让程序算：tight / balance / surplus
    "indicators": {                     // 记录底层趋势性指标，颜色由这些推导
      "inventory_cycle": "底部回补",     // 库存周期位置：去库尾声/底部回补/主动补库/被动累库 等
      "penetration_rate": 35,           // 渗透率 %
      "capacity_utilization": 88,       // 产能利用率 %
      "supply_tightness": "偏紧"         // 供需松紧定性（可选）
    },
    "reasoning": "产能利用率88%偏高+渗透率仍处中段+库存底部回补，三者共振指向紧缺"
  },

  // —— (5) 核心上市公司（至少含龙头）——
  "companies": [
    {
      "name": "生益科技",
      "ticker": "600183.SH",            // 股票代码
      "role": "龙头",                    // 龙头 / 主要参与者
      "purity": 70,                     // 业务纯度：该环节业务占公司营收比 %
      "logic": "全球CCL龙头，高频高速板放量直接受益"  // 一句话受益逻辑
    }
  ],

  // —— (6) 技术演进（可选，有则呈现）——
  "tech_evolution": {
    "stages": ["M6", "M7", "M8", "M9"],
    "current": "M7",
    "note": "M8/M9量产后将全面替代低世代，存在技术迭代逻辑杀风险——价值量高但若被新材料替代则逻辑反转",
    "risk_level": "中"                   // 替代风险：低/中/高
  },

  // —— 上游子节点（递归，按需逐级补全）——
  "upstream": [
    { "id": "electronic_copper_foil", "name": "电子铜箔", "...": "同结构，上游暂可留空待后续精细化" },
    { "id": "insulating_resin", "name": "绝缘树脂", "...": "..." }
  ]
}
```

### 关键设计原则（务必传达给 Codex）

1. **按需展开，不必一次拉满**：新建某环节时，其上游子节点可只填 `id`+`name`，`upstream` 留空；待日后精细化该环节时再补全。程序需优雅处理"数据不完整"的节点（显示为可展开但内容待补）。
2. **价值量占比仅记一级**：`value_share.relative_to` 明确锚点，不做跨级累乘。
3. **缺口颜色由规则推导，不手填结论**：见 §3。

## 3. 缺口状态自动推导规则（投资信号核心）

`gap.status` 不由人工直接填写，而由程序根据 `gap.indicators` 按规则推导，保证判断锚定在可追溯的趋势指标上。建议规则（Codex 可做成可配置的评分函数）：

```
评分维度（各指标映射到 -1 / 0 / +1，+ 代表偏紧）：
- 产能利用率：>85% → +1；70~85% → 0；<70% → -1
- 库存周期：去库尾声/底部回补 → +1；主动补库 → 0；被动累库/主动去库 → -1
- 渗透率：低位且上行(<30%) → +1（成长缺口）；中段(30~60%) → 0；高位(>60%) → -1
- 供需松紧定性：偏紧 +1 / 平衡 0 / 偏松 -1

汇总分 = 各维度求和：
  >= +2  → tight（紧缺，红/橙高亮）
  -1 ~ +1 → balance（平衡，中性色）
  <= -2  → surplus（过剩，蓝/灰）
```

> 规则阈值请做成 JSON 顶部的可配置项（`config.gap_rules`），方便日后调参，无需改代码。
> 同时把推导出的分项贡献显示在详情面板，让我能看到"为什么是红色"。

## 4. 视觉与交互（已确认）

### 4.1 主布局
- **左右横向树状图**：下游在右、上游在左，逐级向左展开，形似思维导图。
- 默认折叠：未点开的上游呈收缩状态（节点旁显示展开标记 ▸），点击展开 ▾。
- 支持画布拖拽平移、滚轮缩放。

### 4.2 节点呈现
节点卡片上直接显示：
- 环节名称（主标题）
- 关键指标缩略：**价值量占比** · **缺口标记** · **龙头股名**（三项缩略，空间有限时缺口用色块、占比用小字、龙头用简称）
- 缺口状态 → **整个节点用颜色高亮**（紧缺=红/橙，平衡=中性，过剩=蓝/灰）

### 4.3 侧边详情面板
点击节点 → 右侧滑出详情面板，完整展示六大维度：
1. 市场规模预测（值/单位/年份/备注）
2. 未来 3-5 年 CAGR
3. 价值量占比（含锚点 relative_to）
4. 缺口状态（颜色结论 + 底层指标 + 推导理由，**显示各指标的评分贡献**）
5. 核心上市公司表格：名称 / 代码 / 角色 / 业务纯度% / 受益逻辑
6. 技术演进：阶段轴（如 M6→M7→M8→M9，高亮 current）+ 替代风险等级 + 说明
- 面板底部显示该环节 `layer_updated` 最后更新日期。

### 4.4 风格
- 简洁、美观、层级化、留白充足。
- 字体清晰，色彩克制（缺口色是唯一强色彩信号，其余保持素雅）。
- 移动端可正常查阅（响应式，树状图在窄屏可切换为可滚动模式）。

## 5. 可拓展性与拼接机制（重点）

这是工具的灵魂：从一个环节起步，逐步拼成 A 股全产业链。

### 5.1 拼接逻辑
- 每个产业链存为独立 JSON（如 `ccl.json`、`pcb.json`）。
- 节点通过 `id` 互相引用。当制作 PCB 时，PCB 的上游之一是 CCL，只需在 PCB 的 `upstream` 中引用 `"ref": "ccl"`，程序自动加载 `ccl.json` 整棵子树挂接进来，实现"逐级打开 CCL 的上游"。
- 这样 CCL 既是独立产业链，又能作为 PCB 的一部分被复用，避免重复维护。

```jsonc
// pcb.json 中引用已有的 ccl
"upstream": [
  { "ref": "ccl" },                     // 直接引用，自动挂接整棵子树
  { "id": "drilling", "name": "钻孔/电镀", "...": "..." }
]
```

### 5.2 索引文件
- 维护一个 `index.json` 列出所有已建产业链及其根节点，作为首页入口导航。
- 首页可选择从哪条产业链进入查看。

## 6. 数据维护工作流（半自动，已确认）

你的工作流：**给研报 → AI 提取 → 生成/合并 JSON → 你 review**。请 Codex 配套实现：

1. **固定的提取 Prompt 模板**：随项目附带一份 `提取模板.md`，里面是给 AI（Claude/Codex）的标准指令——"读这份研报，按 §2 schema 输出该环节的 JSON 片段"。这样每次投喂研报，AI 产出格式统一、可直接合并的 JSON。
2. **校验脚本**：一个本地 JS/Node 脚本，校验 JSON 是否符合 schema（必填字段、ref 是否存在、数值类型），防止脏数据破坏渲染。
3. **数据与代码分离**：所有产业链 JSON 放在 `/data` 目录，改数据不碰代码。
4. 配合 anysearch skill：可在模板中说明"先用 anysearch 检索该环节最新市场规模/龙头数据，再结合我给的研报填充"。

> 数据时效：每个环节记一个 `layer_updated` 即可（已确认，不逐字段记时间戳）。

## 7. 渲染技术建议（给 Codex 的实现提示）

- 树状图推荐 **D3.js 的 tree/hierarchy 布局**（横向），或更轻量的纯 SVG 手绘 + 简单布局算法。D3 成熟、可缩放拖拽、社区方案多，推荐优先。
- 折叠/展开用 D3 的 collapse 模式（点击节点切换 children / _children）。
- 详情面板用原生 DOM 滑出层即可，不必引框架。
- 全部资源本地化或用 CDN，确保离线可用时把 D3 也下载到本地。

## 10. 额外优化建议（供你参考采纳）

1. **"逻辑杀"风险标记**：在节点角标用小图标标出 `tech_evolution.risk_level=高` 的环节，一眼看出"价值量高但有替代风险"的环节——这正是你举 CCL/M7-M9 例子时关心的。
2. **缺口×价值量四象限视图**（可选第二视图）：除树状图外，加一个散点图，X轴=价值量占比、Y轴=缺口紧张度，右上角即"高价值+紧缺"的黄金象限，快速筛标的。
3. **龙头股纯度排序**：详情面板里公司按 `purity` 降序，纯度高的弹性大，符合投资直觉。
4. **搜索框**：环节多了以后，加一个按环节名/公司名/代码的快速搜索定位。
5. **变更留痕**：靠 git 提交记录即可回看每次数据更新，无需额外开发。
6. **导出分享**：预留"导出当前视图为图片/PDF"按钮，方便分享给他人。
