# 研报提取模板（每次更新数据时使用）

> 用途：把研报/资料投喂给 AI（codex 或 Claude），让它按本工具的固定 schema 输出 JSON，可直接合并进 `/data` 目录。
> 用法：把下面"指令正文"连同资料（PDF/截图/文字稿）一起发给 AI 即可。

---

## 指令正文（复制以下全部内容给 AI）

```
你是产业链数据整理助手。请阅读我提供的研报/资料（在本目录或附件中），
按下面的固定 JSON schema，提取并输出指定产业环节的数据。

【输出要求】
1. 严格按 schema 输出，字段名不得改动。
2. 只输出 JSON，不要任何解释文字、不要 markdown 代码块标记。
3. 数据有区间或多口径时，value 取代表值，在 note 里保留区间和口径说明。
4. 找不到的字段填 null，不要编造。
5. 每个数据尽量在 note 里注明来源研报名+日期，便于溯源。
6. gap 使用双轴结构：cycle_axis 填库存周期轴，penetration_axis 填渗透率轴；映射不了的字段填 null。
7. 价值量占比必须注明 relative_to（占谁的比，只算一级，不跨级累乘）。
8. competition 为竞争格局字段，找不到数据时填 null；找到数据时必须包含 global（全球市场地位，含 companies 公司列表）、domestic（国内市场地位，含 companies 公司列表）、key_factors（竞争格局关键成因，字符串数组）三部分。公司列表按市占率从高到低排列，市占率缺失填 null。
9. cycle_axis.stage 只能填 bottom/uptrend/peak/downtrend/null；penetration_axis.stage 只能填 intro/golden/mature/decline/null。
10. 每条轴都要标 source：研报明确阶段或给渗透率数值填 research；按词典/区间推导填 derived。source=research 时 source_ref 必须写“研报名+日期”。
11. 若环节存在技术代际 BOM，必须填写 current_generation 和 bom_by_generation；无代际节点填 current_generation:null、bom_by_generation:[]。
12. bom_by_generation 按每个代际分别列完整 BOM，share 为该材料在该代际中的价值量占比，is_new 标记该代际新出现材料。
13. 材料节点若存在牌号/型号/材料体系，必须填写 grades；BOM item 的 material_id 优先使用 grades[].grade_id，node_id 填所属上游材料节点 id。
14. 公司若有牌号能力信息，必须填写 capability.materials、highest_grade、note；上市状态或代码存疑时 verification_status 填“待核实”，不要臆造代码。
15. 若资料可判断库存周期景气方向，只给 cycle_axis 填 heatmap：按月填写 up/down/flat/null，不需要精确数值；从“持续上涨/去库/需求改善/价格走弱”等定性描述判断方向即可，价格维度尽量必填。
16. penetration_axis 不填 heatmap，改填 trend：按季度填写渗透率数值序列（只填渗透率一个序列），并记录来源；无渗透率概念填 trend:null。默认阈值含义为 0-10% 导入期、10-30% 黄金爆发期、30-50% 成熟分化期、50%+ 衰退/见顶风险区。
17. 若下游只作为需求来源或应用场景锚点、不展开成独立节点，填写 downstream_anchors；若需要进入主图层级，才放入 upstream/独立产业链节点。
18. market_size_forecast 为多来源市场规模预测表（数组），每个元素含 source/base_year/base_value/base_unit/forecast_value/forecast_year/cagr/cagr_period；缺失整字段或某子字段填 null，不要编造数据。
19. growth_drivers 为增长驱动力列表（字符串数组），每条 1-2 句话概括核心驱动力场景与量化逻辑；无资料可填 [] 或 null。


20. demand_analysis 为需求分析九因子星级评定（与 gap 平级），按三层级九因子逐个评估。每个因子给 1-5 星(stars)并写详情(detail)。按以下打分规则执行（此规则仅供AI打分参考，不输出到detail）。

【需求分析 demand_analysis 提取规则】
按三层级九因子逐个评估，每个因子给 1-5 星(stars)并写详情(detail)。
按以下规则打分(此规则仅供你打分参考，不输出到detail)：

广度：
- 渗透率提升[倒U]：<10%极低→低星；10-30%黄金陡峭段→5星；30-60%高增见顶→中星；>60%饱和→低星。详情填当前渗透率%、S曲线位置、近1-2年斜率。
- 应用场景扩张：有新场景且体量大→高星；场景多/单场景大→4-5星；仅小众→2-3星；无→1星。详情填新场景、占比及提升速度。
- 国产替代[倒U]：未启动→低星；加速期(低基数+份额快升)→5星；过半→中星；接近完成→低星。详情填国产化率%、天花板、年提升幅度。

深度：
- 单位用量提升[倒U]：看当前用量vs技术上限倍数空间+增速。放量初期离上限远→5星；接近上限增速放缓→低星。详情填当前单位用量、倍数、还能翻几倍、增速。
- 高端结构升级[视为高端占比的二级渗透率,套倒U]：占比极低刚起步或已成主流→低星；10-30%且快升→5星。详情填高端占比%、ASP提升幅度、斜率。
- 替代升级/技术代际[倒U,新技术渗透率]：刚出未接受→低星；加速替换→高星；替换完成→低星。详情填新技术对旧技术渗透率%、替换速度、驱动力。

持续性：
- 政策/补贴[方向+剩余时间双因子,非倒U]：加码/落地初期→高星；临近退坡→低星，退坡中标负面(透支风险)。详情填政策力度、方向、距退坡时间、透支风险。
- 季节性/行业周期[择时修正,非趋势,偏中性]：底部反转/进旺季→高星；周期顶/旺季末→低星。详情填大周期位置、是否旺季，注明是短期扰动。
- 补库存/囤货[⚠反向,星=虚高/警示强度,越多越危险]：交期拉爆、现货远超原厂涨幅、囤货极端→5星警示；渠道健康→1星。详情填交期周数、渠道库存周数、现货vs原厂涨幅差、囤货惜售迹象。polarity必须填reverse。

输出时 polarity 按框架固定：前8因子除季节性为neutral外均positive，补库存为reverse。
demand_analysis 的 summary 写一句话判读小结（如"供需缺口驱动的周期成长，主引信=单机用量↑+高端占比↑"）。
```

## 你的标准操作流程（SOP）

每次更新数据，按这 5 步：

1. **建文件夹**：在项目 `/updates/` 下新建日期文件夹，如 `/updates/2026-06-25/`。
2. **放资料**：把研报 PDF、网页保存的 HTML/截图、整理的文字稿丢进去。**音频要先转成文字稿**（codex 不能听音频）。
3. **发指令**：把上面"指令正文"复制给 codex，并告诉它：
   > 请读取 `/updates/2026-06-25/` 文件夹里的所有资料，按指令正文的 schema，
   > 提取【环节名】的数据，生成对应的 JSON，并合并/更新到 `/data/对应文件.json`。
   > 完成后告诉我你更新了哪些字段、哪些数据有变化。
4. **review**：检查它产出的 JSON，重点核对市场规模、占比、龙头股有没有offset或编造。
5. **提交**：确认无误后 git commit 一次（codex 可代劳），留下版本记录。

## 几个关键提醒

- **codex 读不了音频**：录音、语音备忘录必须先转文字。手机/电脑都有现成转写工具，转出 .txt 放进文件夹即可。
- **截图可以读**：研报截图、雪球/公众号文章截图，codex 能识别图中文字，但清晰度要够。
- **PDF/Word/Excel 都能读**：直接丢原文件最好，信息最全。
- **一次一个环节**：别一次丢十个产业的资料让它一起做，容易串数据。按环节分批，质量更高。
- **让它报告变化**：每次要求它"告诉我更新了哪些字段、数据有何变化"，你就能快速 review，也能看出趋势（比如缺口是否加剧）。
