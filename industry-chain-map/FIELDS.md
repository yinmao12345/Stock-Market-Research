# FIELDS.md 字段总账

## 总规则

本项目每个字段都同时存在于五处：① 数据 JSON（`/data`）、② schema 定义、③ 数据提取模板、④ 推导/计算规则、⑤ 界面渲染（节点缩略 + 详情面板）。任何字段的增删改，都必须五处同步更新，并同步更新本文档。

本文档是全项目字段定义和缺口评分规则的唯一权威来源。`gap-rules` 等代码必须严格按本文档实现；以后修改本文档中的渗透率、库存周期、产能利用率等规则后，代码、数据模板、schema 和界面说明都必须同步改到一致。

新增产业链只是在现有字段 schema 下填入新的产业数据，不需要逐个产业登记进本文档。本文档只记录字段定义、单位、呈现方式、推导规则和通用流程，不记录具体产业的数据内容。功率半导体数据已按该规则归入半导体产业链，详见 `data/semiconductor.json`。

## 字段登记格式

每个字段条目按以下口径理解：

- 数据 JSON：字段在 `/data/*.json`、`/data/index.json` 或 `/data/config.json` 中的位置。
- schema 定义：字段在项目 schema 中应如何约束。当前项目尚无独立 schema 文件，本文档先作为 schema 的权威来源。
- 数据提取模板：字段在数据采集和更新模板中应如何填写。
- 推导/计算规则：字段是否由程序推导、是否参与评分、阈值如何定义。
- 界面渲染：字段在中间节点缩略和右侧详情面板中的呈现方式。

## 节点基础字段

### `id` / 环节唯一标识

- 含义说明：产业链环节的稳定英文标识，用于树节点定位、引用、搜索和跨文件复用。
- 数据类型与单位：字符串；无单位。建议使用小写英文、数字、下划线或短横线，必须在同一展开后的产业链树内唯一。
- 数据 JSON：位于每个节点对象的 `id`。`data/index.json.chains[].id` 是产业链入口 id。
- schema 定义：节点必填；`ref` 引用节点解析后也必须拥有 `id`。
- 数据提取模板：每个环节必须填写；同一通用环节跨产业复用时保持同一 id。
- 推导/计算规则：不参与缺口评分；`loader` 和树渲染依赖它建立节点身份。
- 界面渲染：详情面板标题区显示为 `ID`；节点搜索匹配 `id`。

### `name` / 环节中文名

- 含义说明：产业链环节在界面上的中文显示名。
- 数据类型与单位：字符串；无单位。
- 数据 JSON：位于每个节点对象的 `name`；`data/index.json.chains[].name` 是产业链入口名称。
- schema 定义：节点必填；索引中的产业链条目必填。
- 数据提取模板：使用行业通用、简洁、可读的中文名称。
- 推导/计算规则：不参与评分。
- 界面渲染：中间画布节点主标题显示；详情面板标题显示；上游节点表显示。

### `layer_updated` / 环节更新时间

- 含义说明：该节点数据最近一次更新的年月或日期。
- 数据类型与单位：字符串；推荐 `YYYY-MM`，必要时可用 `YYYY-MM-DD`。
- 数据 JSON：位于每个节点对象的 `layer_updated`。
- schema 定义：节点推荐填写；缺失时界面显示 `-`。
- 数据提取模板：每次更新节点数据时同步更新。
- 推导/计算规则：不参与评分。
- 界面渲染：详情面板底部显示为最后更新时间。

### `upstream` / 上游环节数组

- 含义说明：当前节点的上游节点列表，构成产业链树结构。
- 数据类型与单位：数组；元素为完整节点对象或包含 `ref` 的引用对象。
- 数据 JSON：位于每个节点对象的 `upstream`。
- schema 定义：节点可选；缺失或空数组表示无上游展开节点。
- 数据提取模板：按产业链上下游关系填写；已有通用环节优先使用 `ref` 复用。
- 推导/计算规则：不参与缺口评分；树渲染用它作为子节点来源。
- 界面渲染：中间画布按 `upstream` 生成可展开树；详情面板上游节点表显示 `name`、`id`、`ref`。

### `ref` / 跨文件引用

- 含义说明：引用已有节点定义，避免在多个产业链中重复维护同一环节。
- 数据类型与单位：字符串；无单位。
- 数据 JSON：常见于 `upstream[]` 元素，例如引用另一个数据文件中的节点。
- schema 定义：引用对象可只含 `ref`，加载阶段应解析为完整节点；解析失败应报错。
- 数据提取模板：新增产业链时，若某上游环节已有权威定义，应优先填写 `ref` 而不是复制完整节点。
- 推导/计算规则：不参与评分；加载器解析引用后，后续计算和渲染使用解析后的完整节点。
- 界面渲染：详情面板上游节点表显示 `ref`；节点缩略不直接显示。

### `downstream_anchors` / 下游锚点

- 含义说明：只作为当前环节需求来源或应用场景锚点的下游清单，不生成独立树节点，适合“下游一级仅作锚点不建节点”的场景。
- 数据类型与单位：数组；元素包含 `name`、`note`。无下游锚点可缺省或填空数组。
- 数据 JSON：位于节点对象顶层，与 `upstream` 同级。
- schema 定义：节点可选字段；`name` 为锚点名称，`note` 为价值量、需求逻辑或场景说明。
- 数据提取模板：当资料只要求标注下游需求来源、不希望扩展独立节点时填写。
- 推导/计算规则：不参与缺口评分，不参与树结构生成。
- 界面渲染：详情面板显示“下游锚点”列表；主图不生成节点。

### `grades` / 材料牌号字典

- 含义说明：材料节点下可供应或可出现在 BOM 中的牌号、材料体系、代际材料字典，用于把 BOM 明细关联到具备该牌号能力的公司。
- 数据类型与单位：数组；元素包含 `grade_id`、`name`、`intro_gen`、`df`、`note`。无牌号节点可缺省或填空数组。
- 数据 JSON：位于材料节点对象顶层，例如 `electronic_copper_foil.grades`。
- schema 定义：节点可选字段；`grade_id` 在同一材料节点内必须唯一；若 BOM item 需要牌号级匹配，应优先使用这里登记的 `grade_id`。
- 数据提取模板：从研报中提取材料牌号、引入代际、关键性能指标和说明。
- 推导/计算规则：不参与缺口评分；UI 用 `grades[].grade_id` 识别 BOM item 所属材料节点，并匹配 `companies[].capability.materials`。
- 界面渲染：当前不单独展示完整字典；在 BOM 行展开牌号龙头时作为匹配依据。

#### `grades[].grade_id` / 牌号标识

- 含义说明：牌号级稳定 id，例如 `hvlp45_foil`、`quartz_cloth`。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中必填；schema 要求稳定唯一；模板按英文小写下划线命名；UI 与 BOM item 的 `material_id`、公司 capability 对齐。

#### `grades[].name` / 牌号名称

- 含义说明：牌号、材料体系或工艺材料中文名。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中必填；schema 要求字符串；模板按资料原名或通用名填写；UI 可用于牌号说明。

#### `grades[].intro_gen` / 引入代际

- 含义说明：该牌号首次成为关键 BOM 材料的技术代际。
- 数据类型与单位：字符串或 `null`；无单位。
- 五处定义/呈现：数据中推荐填写；schema 允许为空；模板按资料提取；UI 可用于理解代际演进。

#### `grades[].df` / 牌号性能指标

- 含义说明：该牌号相关介电损耗或其他性能指标。
- 数据类型与单位：字符串或 `null`；单位随文本自带。
- 五处定义/呈现：数据中可选；schema 允许为空；模板尽量提取；当前 UI 不强制展示。

#### `grades[].note` / 牌号说明

- 含义说明：牌号用途、价格、认证、封装场景或其他补充说明。
- 数据类型与单位：字符串或 `null`；无单位。
- 五处定义/呈现：数据中可选；schema 允许为空；模板记录关键依据；当前 UI 不强制展示。

## 指标字段

### `metrics.market_size` / 市场规模

- 含义说明：该环节对应市场的规模信息。
- 数据类型与单位：对象；由 `value`、`unit`、`year`、`note` 组成。
- 数据 JSON：位于 `metrics.market_size`。
- schema 定义：推荐字段。`value` 为数字或可核验的短文本，`unit` 为字符串，`year` 为年份数字或字符串，`note` 为补充说明。
- 数据提取模板：填写最新可核验口径；必须注明单位和年份，口径差异写入 `note`。
- 推导/计算规则：不参与缺口评分。
- 界面渲染：详情面板顶部指标卡显示；节点缩略当前不显示。

#### `metrics.market_size.value` / 市场规模数值

- 含义说明：市场规模的主数值。
- 数据类型与单位：数字优先；单位由 `unit` 提供。
- 五处定义/呈现：数据中作为原始值；schema 约束为数字优先；模板要求采集值；无推导；详情指标卡拼接展示。

#### `metrics.market_size.unit` / 市场规模单位

- 含义说明：市场规模的计量单位，例如 `亿元`、`亿美元`。
- 数据类型与单位：字符串；字段自身无单位。
- 五处定义/呈现：数据中与 `value` 同时维护；schema 约束为字符串；模板要求必填；无推导；详情指标卡紧跟数值展示。

#### `metrics.market_size.year` / 市场规模年份

- 含义说明：市场规模对应的统计年份。
- 数据类型与单位：数字或字符串；年份。
- 五处定义/呈现：数据中记录口径年份；schema 推荐 `YYYY`；模板要求注明；无推导；详情指标卡展示。

#### `metrics.market_size.note` / 市场规模说明

- 含义说明：市场规模口径、来源、边界或估算说明。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中可选；schema 允许为空；模板用于记录口径差异；无推导；详情指标卡展示。

### `metrics.cagr` / CAGR

- 含义说明：该环节市场规模或相关指标在指定周期内的复合增速。
- 数据类型与单位：对象；由 `value`、`unit`、`period`、`note` 组成。
- 数据 JSON：位于 `metrics.cagr`。
- schema 定义：推荐字段。`value` 为数字优先，`unit` 通常为 `%`，`period` 为时间区间字符串。
- 数据提取模板：必须注明预测或历史周期；口径写入 `note`。
- 推导/计算规则：当前不由程序计算，作为采集字段录入；不参与缺口评分。
- 界面渲染：详情面板顶部指标卡显示；节点缩略当前不显示。

#### `metrics.cagr.value` / CAGR 数值

- 含义说明：复合增速数值。
- 数据类型与单位：数字优先；单位由 `unit` 提供，通常为 `%`。
- 五处定义/呈现：数据中作为原始值；schema 约束为数字优先；模板要求采集；当前无程序推导；详情指标卡拼接展示。

#### `metrics.cagr.unit` / CAGR 单位

- 含义说明：CAGR 的单位。
- 数据类型与单位：字符串；通常为 `%`。
- 五处定义/呈现：数据中维护；schema 推荐 `%`；模板要求注明；无推导；详情指标卡展示。

#### `metrics.cagr.period` / CAGR 周期

- 含义说明：CAGR 对应的统计或预测周期。
- 数据类型与单位：字符串；例如 `2024-2028E`。
- 五处定义/呈现：数据中维护；schema 约束为字符串；模板要求注明；无推导；详情指标卡展示。

#### `metrics.cagr.note` / CAGR 说明

- 含义说明：CAGR 口径、来源、是否预测值等说明。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中可选；schema 允许为空；模板记录来源口径；无推导；详情指标卡展示。

### `metrics.value_share` / 价值量占比

- 含义说明：该环节在指定参照对象中的价值量占比。若父节点存在 `current_generation` 与 `bom_by_generation`，则该值是默认/兜底口径；主图优先使用父节点当前代际 BOM 中该材料的 `share`。
- 数据类型与单位：对象；由 `value`、`unit`、`relative_to`、`note` 组成。
- 数据 JSON：位于 `metrics.value_share`。
- schema 定义：推荐字段。`value` 为数字优先，`unit` 通常为 `%`，`relative_to` 必须说明参照对象。
- 数据提取模板：必须同时填写占比和参照对象，避免不同层级口径混用；有技术代际时还必须填写 `bom_by_generation`，把逐代 BOM 作为权威拆解。
- 推导/计算规则：不参与缺口评分；节点缩略显示值由 UI 在运行时按父节点 `current_generation` 查找 `bom_by_generation[].items[].share`，查不到时回退到本字段。
- 界面渲染：节点缩略显示 `价值量占比：XX%`；详情面板顶部指标卡显示该指标。代际 BOM 的完整拆解在技术演进区展示。

#### `metrics.value_share.value` / 价值量占比数值

- 含义说明：价值量占比的主数值。
- 数据类型与单位：数字优先；单位由 `unit` 提供。
- 五处定义/呈现：数据中作为原始值；schema 约束为数字优先；模板要求采集；无推导；节点缩略和详情指标卡展示。

#### `metrics.value_share.unit` / 价值量占比单位

- 含义说明：价值量占比单位。
- 数据类型与单位：字符串；通常为 `%`。
- 五处定义/呈现：数据中维护；schema 推荐 `%`；模板要求注明；无推导；节点缩略和详情指标卡展示。

#### `metrics.value_share.relative_to` / 占比参照对象

- 含义说明：价值量占比的分母或参照层级。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中维护；schema 应设为必填；模板要求明确填写；无推导；详情面板应展示，但当前未明确展示。

#### `metrics.value_share.note` / 价值量占比说明

- 含义说明：价值量占比口径、估算方法或补充说明。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中可选；schema 允许为空；模板记录口径；无推导；详情指标卡展示。

## 代际 BOM 字段

### `current_generation` / 当前主流技术代际

- 含义说明：该节点当前用于主图取值和详情面板默认选中的技术代际，例如高速 CCL 的 `M8`。
- 数据类型与单位：字符串或 `null`；无单位。
- 数据 JSON：位于每个完整节点对象顶层。无代际节点必须填 `null` 作为占位。
- schema 定义：节点推荐字段；有 `bom_by_generation` 的节点应填写为其中一个 `generation`，无代际节点为 `null`。
- 数据提取模板：从资料中判断当前主流代际；若无法判断但存在 BOM，可填最接近当前出货主流的代际并在 `source_ref` 或 `note` 中说明。
- 推导/计算规则：不参与缺口评分；用于主图子节点占比切片和详情面板默认代际。
- 界面渲染：详情面板代际 BOM 默认选中该代际；主图上游子节点占比优先取该代际 BOM 中匹配 `material_id` 的 `share`。

### `bom_by_generation` / 技术代际 BOM 拆解

- 含义说明：按技术代际记录完整 BOM 价值量拆解，解决价值量占比随代际变化的问题。
- 数据类型与单位：数组；无代际节点填空数组 `[]`。
- 数据 JSON：位于每个完整节点对象顶层，与 `upstream` 解耦。`upstream` 负责拓扑，`bom_by_generation` 负责每代材料占比。
- schema 定义：数组元素包含 `generation`、`df`、`application`、`source_ref`、`items`。`items` 为该代际完整 BOM 清单。
- 数据提取模板：按每个代际逐代提取材料清单、占比、应用场景、来源；不同代际材料清单可不同。
- 推导/计算规则：不参与缺口评分；UI 按 `material_id` 聚合生成趋势速览，并按 `current_generation` 为主图提供当前代际占比。
- 界面渲染：详情面板技术演进区下方显示代际切换、趋势速览、BOM 明细表；`is_new` 项高亮显示。

#### `bom_by_generation[].generation` / 代际名称

- 含义说明：技术代际标识，例如 `M7`、`M8`、`M9`、`M10`。
- 数据类型与单位：字符串；无单位。
- 五处定义/呈现：数据中必填；schema 要求同一节点内唯一；模板按资料填写；用于匹配 `current_generation`；UI 作为切换标签展示。

#### `bom_by_generation[].df` / 代际性能指标

- 含义说明：该代际的介电损耗或其他核心性能指标。
- 数据类型与单位：字符串或 `null`；单位随指标文本自带。
- 五处定义/呈现：数据中可选；schema 允许为空；模板尽量提取；无计算；UI 在代际摘要中展示。

#### `bom_by_generation[].application` / 代际应用场景

- 含义说明：该代际对应的应用、速率、平台或量产阶段。
- 数据类型与单位：字符串或 `null`；无单位。
- 五处定义/呈现：数据中可选；schema 允许为空；模板要求提取；无计算；UI 在代际摘要中展示。

#### `bom_by_generation[].source_ref` / 代际 BOM 来源

- 含义说明：该代际 BOM 数据的资料来源。

## 竞争格局字段

### `competition` / 竞争格局

- 含义说明：该环节的行业竞争结构，包括行业集中度、龙头市占率和竞争梯队描述。

## 缺口字段与双轴评价规则

> 本节已由《缺口双轴评价体系方案》替换原简单 +1/0/-1 缺口规则。`gap` 不再合并为单一 `status/score`，而是拆成 `cycle_axis` 与 `penetration_axis` 两条独立轴。面板只展示阶段定位和来源，不展示交易策略。

### `gap` / 缺口双轴对象

- 含义说明：环节的缺口评价总对象，包含库存周期轴和渗透率轴。
- 数据类型与单位：对象；无单位。
- 数据 JSON：位于每个节点对象的 `gap`。
- schema 定义：必须包含 `cycle_axis`、`penetration_axis` 两个子对象；旧 `gap.status`、`gap.score`、顶层 `gap.breakdown`、`gap.indicators` 不再作为源数据结构使用。
- 数据提取模板：按两轴分别填写阶段、来源、依据和指标，映射不了的字段填 `null` 占位。
- 推导/计算规则：`gap-rules.js` 按本文档词典和渗透率区间分别推导两轴阶段；研报明确值优先于推导值。
- 界面渲染：节点缩略显示两轴阶段；详情面板分两个区块展示阶段、来源、依据，悬停来源显示 `source_ref` 或推导依据。

### `gap.cycle_axis` / 库存周期轴

- 含义说明：供需钟摆位置，回答当前处在库存周期的哪一段。
- 数据类型与单位：对象；无单位。
- schema 定义：包含 `stage`、`source`、`evidence`、`source_ref`、`indicators`、`breakdown`。
- 四阶段：`bottom`=底部；`uptrend`=上行；`peak`=顶峰；`downtrend`=下行。
- 推导/计算规则：若研报明确阶段，则 `source=research` 并直接采用；否则按七维指标词典投票，得票最多的阶段为 `stage`，平票且相邻时可写成过渡阶段如 `uptrend->peak`，并在 `breakdown` 中列明各维度命中。
- 界面渲染：节点缩略以“周期:阶段(来源)”显示；详情面板第一块展示库存周期轴。

### `gap.cycle_axis.indicators` / 库存周期七维指标

- 含义说明：用于推导库存周期阶段的七个定性维度。
- 数据类型与单位：对象；各字段为字符串或 `null`。
- 字段：`inventory`（库存）、`capex`（资本开支）、`demand`（需求）、`price`（价格）、`utilization`（开工率）。
- 数据提取模板：尽量填研报原文或归纳短句；没有资料时填 `null`。
- 推导/计算规则：每个非空维度按本文档权威词典匹配到 `bottom/uptrend/peak/downtrend` 之一，形成投票。
- 界面渲染：详情面板可在 `breakdown` 中展示维度命中的阶段。

### `gap.penetration_axis` / 渗透率轴

- 含义说明：成长生命周期位置，回答新技术、新材料、新工艺或新场景的渗透阶段。
- 数据类型与单位：对象；`rate` 单位为 `%`，数据中只填数值。
- schema 定义：包含 `stage`、`source`、`evidence`、`source_ref`、`rate`、`type`、`indicators`、`breakdown`、`trend`；不再使用 `penetration_axis.heatmap`。
- 四阶段：`intro`=导入期（0-10%）；`golden`=黄金爆发期（10-30%）；`mature`=成熟分化期（30-50%）；`decline`=出清/衰退期（50%+ 或见顶/被替代）。
- 四类型：`structural`=结构替代型；`space_creating`=空间创造型；`supply_chain`=供应链自主型；`scenario`=场景跨界型。
- 推导/计算规则：若研报给出渗透率数值，则按区间直接归类并标 `source=research`；若只有定性描述，则按特征词推导并标 `source=derived`。`trend.values` 的最新值可作为 `rate` 的权威来源，最终 `stage` 允许人工确认覆盖。
- 界面渲染：节点缩略以“渗透:阶段(来源)”显示；详情面板第二块展示渗透率、类型、依据和带阈值带的趋势折线图，不展示热力图。

### `gap.*.source` / 来源双轨

- 含义说明：标注每条轴的阶段来自研报明确结论还是规则推导。
- 数据类型与单位：枚举字符串；允许值为 `research`、`derived`，未知时为 `null`。
- 数据 JSON：位于 `gap.cycle_axis.source` 与 `gap.penetration_axis.source`。
- 数据提取模板：研报直接说阶段或给渗透率数值时填 `research`；由词典或区间推导时填 `derived`。
- 推导/计算规则：`research` 优先级高于 `derived`。`source=research` 时 `source_ref` 必填“研报名+日期”。
- 界面渲染：阶段标签旁显示“研报/推导”小标记，悬停显示 `source_ref` 或 `evidence`。

### `gap.*.breakdown` / 推导拆解

- 含义说明：规则推导时各维度或区间命中的阶段，用于溯源。
- 数据类型与单位：数组；元素至少包含 `dimension`、`value`、`stage`。
- 数据 JSON：位于 `gap.cycle_axis.breakdown` 与 `gap.penetration_axis.breakdown`。
- schema 定义：可为空数组；当 `source=derived` 时建议填写。
- 数据提取模板：人工直接给出 `research` 阶段时可为空；推导时列出命中依据。
- 推导/计算规则：由 `gap-rules.js` 根据七维词典或渗透率区间生成/补全。
- 界面渲染：详情面板按行展示维度和命中阶段。

## 新增产业链的标准流程（SOP）

新增产业链是按现有字段 schema 填写新的产业数据，不需要把每个产业逐个登记进本文档。本文档只维护字段定义、字段规则和通用流程。

1. 新建产业链数据文件：在 `/data` 下创建新的 `{chain_id}.json`，根节点必须包含 `id`、`name`、`metrics`、`competition`、`gap`、`companies`、`tech_evolution`、`upstream`、`layer_updated` 等现有字段结构。
2. 复用已有环节：如果新增产业链中的上游或共用环节已在其他数据文件中存在，优先用 `ref` 引用已有环节，避免复制粘贴造成多处不一致。
3. 更新索引：在 `/data/index.json` 的 `chains` 中新增 `{ id, name, note }`；如需默认打开该产业链，再更新 `default_chain`。
4. 按总账 schema 填写字段：所有字段的类型、单位、必填/可选口径以本文档为准。市场规模必须带单位和年份，CAGR 必须带周期，价值量占比必须带参照对象；存在技术代际时必须填写 `current_generation` 和 `bom_by_generation`，无代际节点填 `current_generation:null`、`bom_by_generation:[]`；竞争格局找不到资料时填 `null`。
5. 使用双轴缺口结构：`gap.cycle_axis` 填库存周期阶段和七维指标，`gap.penetration_axis` 填渗透率阶段、rate、type 和依据；映射不了的字段填 `null`。
6. 不手填旧运行时字段：不要在源数据中填写旧 `gap.status`、`gap.score` 或顶层 `gap.breakdown`。
7. 写清双轴依据：研报明确值填 `source=research` 和 `source_ref`；推导值填 `source=derived`、`evidence` 和 `breakdown`。
8. 更新五处来源：如果新增产业链过程中发现需要新增字段，必须同步更新 `/data`、schema、数据提取模板、推导/计算规则、界面渲染，并更新本文档。
9. 完成校验：新增后检查 JSON 可加载、节点可展开、详情面板字段完整、缺口评分明细符合本文档规则。
